🎤 Prompt for Replit AI: Fix Microphone Permission and Recording Logic

The microphone for the DreamScape AI chat (Sleeping category) still doesn’t work properly.

Symptoms:

It asks for microphone permission immediately on entering the category, even before pressing the mic button.

When I press the mic, it says “Could not start recording. Please ensure microphone permissions are enabled,” even though I already allowed access.

Fix this by restructuring the microphone logic as follows:

Ask for microphone permission only when the mic button is first pressed, not on page load.

Remove any automatic permission requests in component mount or app startup.

Move permission request inside the mic button handler.

Initialize the voice recorder or speech recognizer after permission is granted.
Example pattern (Capacitor VoiceRecorder):

import { VoiceRecorder } from 'capacitor-voice-recorder';
import { useState } from 'react';

const MicButton = ({ onResult }) => {
  const [recording, setRecording] = useState(false);

  const handleMicClick = async () => {
    try {
      const perm = await VoiceRecorder.hasPermission();
      if (!perm.value) {
        const request = await VoiceRecorder.requestPermission();
        if (!request.value) {
          alert('Microphone permission denied');
          return;
        }
      }

      if (!recording) {
        await VoiceRecorder.startRecording();
        setRecording(true);
      } else {
        const result = await VoiceRecorder.stopRecording();
        setRecording(false);
        if (result.value && result.value.recordDataBase64) {
          // send audio to backend or speech-to-text
          onResult(result.value.recordDataBase64);
        }
      }
    } catch (err) {
      console.error('Mic error:', err);
      alert('Could not start or stop recording. Check microphone access.');
    }
  };

  return (
    <button onClick={handleMicClick} className={recording ? 'mic-active' : 'mic'}>
      🎤
    </button>
  );
};


Ensure Info.plist permissions exist in the iOS project:

<key>NSMicrophoneUsageDescription</key>
<string>Microphone access is needed for voice input in DreamScape AI.</string>


Check for active session conflicts.

Before starting a new recording, always call VoiceRecorder.stopRecording() if recording is already true.

Add a small delay (200–300ms) between stopping and starting again.

If using Web Speech API instead of Capacitor:

Ensure recognition.start() is only called after the permission prompt resolves.

Wrap it in a try/catch to handle “network” or “not allowed” errors gracefully.
Example:

try {
  recognition.start();
} catch (e) {
  console.error('SpeechRecognition error:', e);
  recognition.abort();
}


Visual feedback:

Show a glowing or pulsing icon when the mic is active.

Disable the button briefly while starting or stopping to prevent double-tap race conditions.

Testing checklist:

Enter “Sleeping” → no permission prompt yet.

Press mic → permission request appears once.

After granting → starts recording immediately.

Press mic again → stops cleanly and sends audio for transcription.

No more “Could not start recording” errors.

🧠 Coaching Notes for Replit AI

The key fix is delaying permission and initialization until the mic button is pressed.

Use a single state variable (recording) to track session status and avoid overlapping recordings.

Always include the Info.plist microphone key or iOS will silently fail even if permission appears granted.

Add a try/catch around both startRecording() and stopRecording() calls.

If using Capacitor, call VoiceRecorder.requestPermission() only when hasPermission() is false.
