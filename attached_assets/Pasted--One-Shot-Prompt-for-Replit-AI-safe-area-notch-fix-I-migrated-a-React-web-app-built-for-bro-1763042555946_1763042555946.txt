I migrated a React web app (built for browser) into a native iOS app using Capacitor/CocoaPods. Titles are rendering under the notch. Fix safe-area handling across the app with these exact changes:

1) index.html (viewport)

Ensure the viewport tag is EXACTLY this (replace if different):

<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no, viewport-fit=cover">


2) Global CSS (create or update src/styles/safe-area.css and import it once in the app root)

Add safe-area CSS variables and helpers:

:root {
  /* Fallbacks for non-notch devices */
  --safe-top: 0px;
  --safe-bottom: 0px;
  --safe-left: 0px;
  --safe-right: 0px;
}

/* iOS notch support */
@supports (padding-top: env(safe-area-inset-top)) {
  :root {
    --safe-top: env(safe-area-inset-top);
    --safe-bottom: env(safe-area-inset-bottom);
    --safe-left: env(safe-area-inset-left);
    --safe-right: env(safe-area-inset-right);
  }
}

/* Common layout helpers */
.safe-page {
  /* app background behind the notch */
  background: #000; /* or your app bg */
  min-height: 100dvh; /* modern viewport unit */
  padding-top: var(--safe-top);
  padding-bottom: var(--safe-bottom);
}

/* Fixed/Sticky headers should sit below the notch */
.safe-header {
  position: sticky;
  top: 0;
  padding-top: var(--safe-top);
  background: inherit;
  z-index: 1000;
}

/* If you use position:fixed headers */
.safe-header-fixed {
  position: fixed;
  left: 0;
  right: 0;
  top: 0; /* visual top */
  padding-top: var(--safe-top);
  z-index: 1000;
}

/* Body scroll bounce + overscroll behavior for iOS */
html, body, #root {
  height: 100%;
  overscroll-behavior-y: none;
  -webkit-overflow-scrolling: touch;
  background: #000; /* match app bg to avoid white notch strip */
}


Import this once (e.g., in src/main.tsx or src/index.tsx):

import './styles/safe-area.css';


Ensure each top-level page/container wraps content with className="safe-page" and headers use safe-header or safe-header-fixed.

(async () => {
